# SoulDance 后端启动指南

## 架构说明

```
┌─────────────┐      USB      ┌──────────────┐     SSH/网络     ┌─────────────┐
│   手机(App) │ ──adb reverse─→│  本地电脑     │ ────隧道─────→│ 远程服务器   │
│  localhost  │                │ localhost    │                │ 0.0.0.0:8000│
│   :8000     │                │   :8000      │                │  (后端服务)   │
└─────────────┘                └──────────────┘                └─────────────┘
```

- **远程服务器**：通过 SSH 连接（`efotf1lm.zjz-service.cn:22549`），运行后端服务
- **本地电脑**：Windows/Mac，手机通过 USB 插在这台电脑上
- **手机**：通过 adb reverse 映射到本地电脑，再通过 SSH 隧道访问远程服务器

---

## 快速启动（3 步）

### 第 1 步：在远程服务器启动后端

SSH 连接到服务器，进入项目目录：

```bash
cd ~/houlong/SoulDance
./start_backend.sh
```

启动成功后，你会看到：

```
Listen:   http://0.0.0.0:8000
Health:   http://0.0.0.0:8000/health
WS Chat:  ws://0.0.0.0:8000/ws/chat
```

**这个终端不要关。**

> 💡 `start_backend.sh` 已增加自动清理逻辑，如果 8000 端口被旧进程占用，会自动杀掉 adb-bridge 的残留进程。

---

### 第 2 步：在本地电脑建立 SSH 端口转发

**另开一个终端**（本地电脑的 PowerShell / CMD / Git Bash），运行：

```bash
ssh -L 8000:localhost:8000 -p 22549 huadabioa@efotf1lm.zjz-service.cn -N
```

这会把本地电脑的 `localhost:8000` 映射到远程服务器的 `localhost:8000`。

**这个终端也不要关。**

如果想简化，可以在 `~/.ssh/config` 里加配置：

```
Host mix_A100_tunnel
    HostName efotf1lm.zjz-service.cn
    User huadabioa
    Port 22549
    IdentityFile ~/.ssh/mix_A100_ed25519
    LocalForward 8000 localhost:8000
```

然后只需要运行：

```bash
ssh mix_A100_tunnel -N
```

---

### 第 3 步：在本地电脑配置 adb

**再开一个终端**，运行：

```bash
adb reverse tcp:8000 tcp:8000
```

然后手机 App 配置地址：

```
http://localhost:8000
```

## 踩坑记录 & 解决方案

### ❌ 坑 1：在服务器上误运行 `adb-bridge.sh`

**现象**：服务器上运行 `adb-bridge.sh` 后，Python 进程占用了 8000 端口，导致后端启动失败（`address already in use`）。

**原因**：`adb-bridge.sh` 是设计在**本地电脑**上运行的。它内部的 Python 端口转发会监听 0.0.0.0:8000，与后端服务冲突。

**解决**：
- `adb-bridge.sh` 已增加 SSH 环境检测，在远程服务器上运行会自动退出并提示。
- `start_backend.sh` 已增加自动清理逻辑，启动前会检测并杀掉 adb-bridge 的残留进程。

### ❌ 坑 2：Python 环境找不到

**现象**：`adb-bridge.sh` 报错 `python3: command not found`。

**原因**：脚本默认查找项目虚拟环境 `env/venv_vllm_cu128` 中的 Python，如果找不到则回退系统 PATH。

**解决**：脚本已增强 Python 路径检测，支持 `VENV_DIR` 环境变量覆盖。

### ❌ 坑 3：`adb-bridge.sh` 里的 `{SERVER}` 变量未替换

**现象**：Python 报错 `NameError: name 'SERVER' is not defined`。

**原因**：脚本中一处 f-string 里的 `{SERVER}` 缺少 `$` 前缀，shell 没有替换变量。

**解决**：已修复为 `$SERVER`。

## 验证检查清单

启动完成后，逐项确认：

- [ ] 远程服务器：`./start_backend.sh` 正常运行，显示 `Uvicorn running`
- [ ] 远程服务器：`lsof -i :8000` 显示 uvicorn 进程在监听
- [ ] 本地电脑：`ssh -L ...` 命令运行中，没有报错
- [ ] 本地电脑：`adb reverse --list` 显示 `tcp:8000 tcp:8000`
- [ ] 手机：App 配置 `http://localhost:8000`
- [ ] 手机：能正常访问后端接口

## 脚本说明

| 脚本 | 运行位置 | 作用 |
|------|---------|------|
| `start_backend.sh` | 远程服务器 | 启动后端服务（自动清理端口占用） |
| `adb-bridge.sh` | 本地电脑 | 手机桥接到远程服务器（含 SSH 环境检测） |
| `local-bridge.sh` | 本地电脑 | 替代方案：Python 端口转发（无需 ssh -L） |
