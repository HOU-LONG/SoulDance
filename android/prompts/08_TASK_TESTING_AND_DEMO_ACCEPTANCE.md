# Task 08：Android 端测试、验收与 Demo 彩排

## 目标

确保 Android 客户端稳定、可演示、不卡顿、不崩溃。

## 必跑 Demo Query

```text
1. 推荐一款适合油皮的洗面奶，预算100以内
2. 推荐防晒霜，但不要含酒精的，也不要日系品牌
3. 下周去三亚度假，帮我搭配一套从防晒到穿搭的方案
4. 点开主推商品，输入：这个有点贵，有没有100以内的？
5. 点开主推商品，输入：这个适合敏感肌吗？
6. 把第一款加入购物车，把数量改成2
7. 看看购物车，然后下单
```

## Android 端验收矩阵

| 模块 | 验收项 |
|---|---|
| Chat UI | 页面美观，消息不拥挤，输入栏固定 |
| WebSocket | 能连接、断线不崩溃、错误可提示 |
| Text streaming | text_delta 能逐字显示 |
| Product cards | skeleton 后逐张出现，主推商品突出 |
| Product focus | BottomSheet 能打开，followup 能发送 |
| Replacement | 替代商品能在详情浮层中出现 |
| Bundle | 三亚方案能按分组展示 |
| Cart | 加购、删除、数量编辑、合计、结算 |
| ASR | 真机能识别中文 |
| TTS | audio_delta 能播放，可停止 |
| Lifecycle | 页面销毁释放 WebSocket/SpeechRecognizer/AudioTrack |
| Error handling | 后端失败、图片失败、TTS 失败都不崩溃 |
